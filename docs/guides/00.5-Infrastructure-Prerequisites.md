# TrueNAS Pool Architecture

TrueNAS at `10.0.10.80` uses three ZFS pools, each with a distinct purpose:

| Pool | Purpose |
|---|---|
| `tera` | Bulk cold storage — large media files, archives, rarely-accessed data |
| `core` | Hot storage — Kubernetes persistent volumes, actively read/written by the cluster |
| `archive` | Backup storage — etcd snapshots, Velero data via MinIO, host backups |

All datasets on these pools are mounted at `/mnt/<pool>/<dataset>` on TrueNAS, which is the path
used when configuring NFS exports and host paths.

```
TrueNAS ZFS Layout
├── core/
│   └── k8s-volumes/          ← Kubernetes PVCs (provisioned dynamically by NFS provisioner)
├── archive/
│   └── backups/              ← NFS-exported backup parent
│       ├── k8s/              ← Kubernetes cluster backups
│       │   ├── etcd/         ← k3s etcd snapshots
│       │   └── minio/        ← MinIO object store data (used by Velero)
│       ├── docker/           ← Docker host appdata backups
│       └── rpi/              ← Raspberry Pi key material backups
└── tera/
    ├── media/                ← Movies and TV series
    └── downloads/            ← In-progress downloads (staging)
```

# Step 1 — Create the Kubernetes Volumes Dataset

This dataset backs all Kubernetes PersistentVolumeClaims. The NFS provisioner will create
one subdirectory per PVC automatically.

**In the TrueNAS web UI:**

1. Navigate to **Storage → Create Dataset**
2. Parent: `core`
3. Name: `k8s-volumes`
4. Share Type: **Generic**
5. Click **Save**

**Or via SSH on TrueNAS:**

```bash
zfs create core/k8s-volumes
```

Verify:

```bash
zfs list core/k8s-volumes
# Should show: core/k8s-volumes with mount point /mnt/core/k8s-volumes
```

---

# Step 2 — Create the Backup Datasets

`archive/backups` is the NFS-exported parent. All backup destinations are nested within it.

**Or via SSH on TrueNAS:**

```bash
# Parent backup dataset (NFS exported)
zfs create archive/backups

# Kubernetes cluster backups
zfs create archive/backups/k8s
zfs create archive/backups/k8s/etcd
zfs create -o compression=off -o sync=disabled -o recordsize=1M archive/backups/k8s/minio

# Host backups
zfs create archive/backups/docker
zfs create archive/backups/rpi
```

> MinIO sync is disabled deliberately — MinIO has its own internal consistency mechanisms.

Verify:

```bash
zfs list | grep backups
# Expected:
# archive/backups                         ...  /mnt/archive/backups
# archive/backups/k8s             ...  /mnt/archive/backups/k8s
# archive/backups/k8s/etcd        ...  /mnt/archive/backups/k8s/etcd
# archive/backups/k8s/minio       ...  /mnt/archive/backups/k8s/minio
# archive/backups/docker          ...  /mnt/archive/backups/docker
# archive/backups/rpi             ...  /mnt/archive/backups/rpi
```

# Step 3 — Configure NFS Exports

Two NFS shares are required. Both are exported to the cluster node subnet only.

**In the TrueNAS web UI, navigate to Sharing → NFS → Add.**

### Share 1 — Kubernetes Volumes

| Setting | Value |
|---|---|
| Path | `/mnt/core/k8s-volumes` |
| Maproot User | `root` |
| Maproot Group | `wheel` |
| Allowed Networks | `10.0.10.0/24` |
| Enable NFSv4 | Yes |

### Share 2 — Backup Storage

| Setting | Value |
|---|---|
| Path | `/mnt/archive/backups` |
| Maproot User | `root` |
| Maproot Group | `wheel` |
| Allowed Hosts | `10.0.10.11` (tywin), `10.0.10.31` (staging-k3s VM), `10.0.10.32` (Docker VM), `10.0.10.10` (bran/RPi) |
| Enable NFSv4 | Yes |

After saving each share, restart the NFS service:

**System → Services → NFS → Restart**

### Verify NFS Exports are Active

From the automation host (`bran`, 10.0.10.10):

```bash
showmount -e 10.0.10.80
```

Expected output:

```
Export list for 10.0.10.80:
/mnt/core/k8s-volumes    10.0.10.0/24
/mnt/archive/backups     10.0.10.10 10.0.10.11 10.0.10.31 10.0.10.32
```

If `showmount` is not installed:

```bash
sudo apt install -y nfs-common
```

# Step 4 — Create the Cloudflare API Token

cert-manager uses a Cloudflare API token to complete the DNS-01 ACME challenge when
issuing the `*.kagiso.me` wildcard TLS certificate. The token is stored as an Ansible Vault
secret and injected into the cluster during the networking platform setup (Guide 03).

**Required permissions:**

| Zone | Permission |
|---|---|
| Zone → Zone | Read |
| Zone → DNS | Edit |

**Scoped to:** `kagiso.me` only (not all zones).

**Steps:**

1. Log into the Cloudflare dashboard
2. Navigate to **My Profile → API Tokens → Create Token**
3. Start from the **Edit zone DNS** template
4. Set **Zone Resources** to: `Include → Specific zone → kagiso.me`
5. Click **Continue to summary → Create Token**
6. **Copy the token now** — it is only shown once. Store it in your password manager.

The token will be placed into `ansible/vars/vault.yml` in Guide 03.
